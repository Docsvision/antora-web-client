= Работа с делами номенклатуры дел

// WARNING: Дела не могут быть созданы в _системном_ разделе _Разделы_ и в _корневых_ xref:directories/nomenclature/.sections.adoc[разделах].
//
// Для работы xref:docs-to-case.adoc[функции списания] дел из карточки, администратору системы {dv} необходимо настроить карточки документов, xref:layouts:guide-add-case-control.adoc[добавив] необходимый элемент управления в разметку.
//
// Посмотреть информацию об уже созданном деле можно из контекстного меню дела -- пункт _Информация_ или по клику на само дело.
//
// .Окно просмотра информации о деле
// image::case-info.png[Окно просмотра информации о деле]
//
// В окне информации о деле отображаются _Сведения о родительском разделе_, _Сведения о деле_ и _Документы, списанные в дело_.
//
// В блоке _Документы, списанные в дело_ отображаются все документы, которые xref:docs-to-case.adoc[были списаны в дело]. Документы отображаются как ссылки на карточки, при нажатии на ссылку в новой вкладке открывается карточка документа.
//
// Ниже расположены кнопки, позволяющие перевести дело в другое состояние. В зависимости от текущего <<caseStates,состояния дела>> и xref:directories/nomenclature/.security.adoc[настроек безопасности], дела и кнопки перевода дела в другое состояние могут быть недоступны пользователю.
//
// == Жизненный цикл дела
//
// Жизненный цикл дела регламентирован состояниями и переходами из одного состояния в другое. От текущего состояния дела зависит возможность xref:docs-to-case.adoc[списывать документы в дело] или <<editCase,изменять>> дело. В таблице описаны состояния и возможные переходы между ними.
//
// [#caseStates]
//
// [plantuml,svg]
// ....
// @startuml
// control Проект as Foo2
// control Утверждено as Foo3
// control Закрыто as Foo4
// entity "Списать в дело" as Foo5
// entity "Удалить документ" as Foo6
// entity "Удалить дело" as Foo7
// entity "Прочесть, \nизменить дело" as Foo8
//
// alt#yellow #gold Из состояния Проект ответственные за дело могут
// Foo2 -> Foo3: Утвердить
// note right
// В состоянии Утверждено всегда создаются новые года.
// end note
// Foo2 -> Foo4: Закрыть
// Foo2 -> Foo6: Удалять документы из дела
// Foo2 -> Foo7: Удалять дела
// Foo2 -> Foo8: Читать или изменять дела
// end
// alt#green #orange Из состояния Утверждено ответственные за дело могут
// Foo3 -> Foo8: Читать или изменять дела
// Foo3 -> Foo6: Удалять документы из дела
// Foo3 ->x Foo7: Удалять дела
// Foo3 -> Foo2: Вернуть в проект
// note right
// В состоянии Проект всегда создаются новые дела.
// end note
// Foo3 -> Foo4: Закрыть
// end
// alt#purple #pink Из состояния Закрыто ответственные за дело могут
// Foo4 -> Foo8: Читать или изменять дела
// Foo4 -> Foo6: Удалять документы из дела
// Foo4 ->x Foo7: Удалять дела
// Foo4 -> Foo2: Вернуть в проект
// Foo4 -> Foo2: Вернуть в утверждено
// end
// alt#lightblue #cyan Списание документов в дело
// Foo5 ->x Foo2: Запрещено списывать документы в дела в состоянии Проект
// Foo3 -> Foo2: Списать документ в дело
// Foo4 -> Foo2: Списать документ в дело
// end
// @enduml
// ....
//
// .Подробнее о состояниях и переходах в таблице:
// [%collapsible%open]
// ====
// .Состояния дела и переходы из состояний
// [cols="34%,33%,33%", options="header"]
// |===
// |Состояние
// |Описание
// |Доступные переходы
//
// |Проект
// |Пользователи, xref:directories/nomenclature/.sections.adoc#responsibleEmployees[ответственные] за дело могут читать, <<editCase,изменять>> и <<deleteCase,удалять>> дело или удалять документы из дела.
//
// Функция списания документов в дело недоступна.
//
// В состоянии `Проект` всегда создаются новые дела.
// |Утвердить, Закрыть
//
// |Утверждено
// |Пользователи, xref:directories/nomenclature/.sections.adoc#responsibleEmployees[ответственные] за дело могут читать и <<editCase,изменять>> дело, удалять документы из дела, но не могут удалять дело.
//
// Функция списания документов в дело доступна.
//
// В состоянии `Утверждено` всегда создаются новые года.
// |Вернуть в Проект, Закрыть
//
// |Закрыто
// |Пользователи, xref:directories/nomenclature/.sections.adoc#responsibleEmployees[ответственные] за дело могут читать и <<editCase,изменять>> дело, удалять документы из дела, но не могут удалять дело.
//
// Функция списания документов в дело доступна.
// |Вернуть в Проект, Вернуть в Утверждено
// |===
// ====
//
// [#createCase]
// == Создание нового дела
//
// Дела создаются в состоянии `Проект`.
//
// .Чтобы создать новое дело:
// . Выберите раздел для создания дела.
// include::partial$excerpts.adoc[tags=createMethods]
// . Заполните <<caseFields,поля>> карточки дела, затем нажмите *Создать*.
// +
// .Окно создания нового дела
// image::new-case.png[Окно создания нового дела]
//
// Теперь вы можете начать xref:docs-to-case.adoc[списывать] документы в дело.
//
// [#editCase]
// == Изменение дела
//
// После создания дела, его можно редактировать при наличии соответствующих xref:directories/nomenclature/.security.adoc[прав доступа]. +
// Редактировать дела можно только в состоянии `Подготовка`.
//
// .Чтобы изменить дело:
// . Выберите раздел, в котором хранится дело.
// . Перейдите в область дел, вызовите контекстное меню.
// . Выберите _Редактировать_.
// . Измените <<caseFields,поля>> карточки дела.
// . Сохраните изменения, нажав кнопку *Сохранить*.
//
// [#deleteCase]
// == Удаление дела
//
// Удаление возможно только при наличии соответствующих xref:directories/nomenclature/.security.adoc[прав доступа].
//
// .Чтобы удалить дело:
// . Выберите раздел, в котором хранится дело.
// . Перейдите в область дел, вызовите контекстное меню.
// . Выберите _Удалить_.
// +
// [WARNING]
// ====
// Разделы удаляются со всеми дочерними элементами, включая дела. При удалении дела, списанные в него документы не удаляются. Если в документе была ссылка на дело, такая ссылка будет удалена.
// ====
//
// [#caseFields]
// == Перечень полей карточки дела
//
// У каждого дела есть индекс и заголовок.
//
// [horizontal]
// Заголовок::
// Имя дела, которое будет отображаться в справочнике {dv}.
//
// Индекс::
// Индекс дела формируется автоматически на основании xref:directories/nomenclature/.sections.adoc#index[индекса] родительского раздела. Дела сортируются по индексу в алфавитном порядке от меньшего к большему.
